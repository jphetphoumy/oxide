# ADR-0007: Prefix Dust conversation titles in the CLI

**Date**: 2026-06-02
**Status**: Accepted

## Context

When Oxide creates a new Dust conversation, the title is visible in the Dust web app.
The official Dust CLI sets that title itself instead of relying on Dust to infer it.

During integration, we confirmed the title is not auto-generated by Dust. If Oxide does not
send a title, the conversation appears without the `CLI Question:` prefix in the web UI.

## Decision

When Oxide creates a new conversation from a fresh prompt, set the conversation title in the
request body to:

```text
CLI Question: <first 30 chars of the message>...
```

If the message is 30 characters or shorter, omit the trailing ellipsis.

Oxide only applies this title when starting a new conversation. Follow-up messages keep the
existing conversation title unchanged.

## Consequences

- New conversations created by Oxide appear in Dust with the same CLI-style title prefix as the official client
- The web app clearly distinguishes CLI-created conversations from manually created ones
- The title format is now part of Oxide behavior and should be preserved unless we explicitly choose a new convention
- Existing conversations are unaffected
