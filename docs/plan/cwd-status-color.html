Colorize the current working directory in the TUI status line

Add a visible, accent-colored cwd label to the bottom status bar so the operator can tell which folder the session is tied to without reading logs or shell state.

Current behavior The bottom status line shows agent:, streaming state, and shortcuts, but no working-directory context.
Desired behavior Render the cwd in a distinct color, keep the rest of the line intact, and avoid noisy formatting changes elsewhere.
Main files src/app.rs, src/main.rs, src/ui/layout.rs, docs/plan/cwd-status-color.html

Behavior Contract

  • The app captures the process cwd once at startup and stores it on App.
  • The status line renders the cwd with the accent color, not the dim agent or hint colors.
  • The existing agent label, streaming indicator, and shortcut hints remain visible.
  • Long paths should degrade gracefully if they exceed the available width.

Implementation Steps

  1. Add a cwd field and accessor to App, plus tests for initialization.
  2. Read std::env::current_dir() in run_tui() and pass the value into App::new.
  3. Update render_layout() to build a colored cwd segment ahead of the agent label.
  4. Add a small helper to format the path for display, including a friendly home-directory prefix if available.
  5. Run formatting and tests, then validate the status bar in a terminal session.

Test Plan

Check Expectation
cargo fmt Formatting stays consistent after the new field and rendering changes.
cargo test App and layout tests cover the cwd state and any path-formatting helper.
tmux capture Status line shows the cwd in color on a real terminal, not just in unit tests.

Risks

Width pressure: if the cwd is long, the status line can crowd out the hints. The display helper should abbreviate or clip before that happens.
  • The change touches the shared layout code, so a small formatting mistake can affect every TUI frame.
  • If cwd is resolved differently between shells, the display should still reflect the process directory, not the shell prompt.

Acceptance Criteria

  • OK The status line contains a colored cwd segment.
  • OK The existing agent and shortcut text still fit and render correctly.
  • OK Unit tests cover the new state and formatting logic.
  • OK The UI is verified in a real terminal capture.
