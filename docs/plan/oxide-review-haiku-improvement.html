🚀 Oxide Review Skill - Haiku Exploration Improvement Plan

Objective: Improve the oxide-review skill to leverage haiku-4.5 sub-agents for all codebase exploration tasks, enabling faster, more efficient code analysis while reserving the main Claude session for synthesis, finding prioritization, and review writing.

Current State Analysis

The current oxide-review skill performs all analysis in the main Claude session:

  • Reads plan documents, ADRs, and changed files directly
  • Runs git status, git diff, and test commands
  • Analyzes code for bugs, regressions, and plan mismatches
  • Writes findings to feedback.md

Limitation: All exploratory tasks (file reading, git commands, test execution) consume the main session's context budget, reducing capacity for complex analysis and synthesis.

Proposed Architecture

Two-Agent Pattern:
  • Main Claude (Sonnet/Opus): Orchestrates review, synthesizes findings, writes feedback.md, makes judgment calls
  • Sub-Agent (Haiku): Performs all file reads, git commands, test runs, and preliminary analysis

Responsibilities Division

Task Primary Agent Rationale
Read plan documents (docs/plan/*.html, docs/adr/*.md) Haiku (sub-agent) Fast extraction of acceptance criteria
Run git status, git diff commands Haiku (sub-agent) Efficient file enumeration
Read changed files with context Haiku (sub-agent) Can read large files efficiently
Run cargo test, tmux smoke tests Haiku (sub-agent) Lightweight execution, reports pass/fail
Preliminary code analysis Haiku (sub-agent) Flags obvious issues, summarizes findings
Validate and prioritize findings Main Claude Needs full context, makes judgment calls
Write feedback.md Main Claude Final arbiter of findings and tone

2. Haiku Sub-Agent Tasks

The Haiku sub-agent will accept these exploration requests:

Exploration Request Format: 
{
  type: "explore",
  task: "read-plan" | "read-diffs" | "read-files" | "run-tests" | "analyze-code",
  params: {
    planPath?: string,           // e.g., "docs/plan/my-feature.html"
    filePaths?: string[],        // e.g., ["src/main.rs", "src/config.rs"]
    testType?: "unit" | "smoke", // default: "unit"
    focus?: string               // e.g., "memory leaks" | "API compatibility"
  }
}

3. Main Skill Workflow

Revised Review Workflow (using Haiku sub-agent):
  1. Parse user request → identify plan document / scope
  2. Call Haiku sub-agent with "read-plan" task
    • Sub-agent reads plan/ADR, extracts acceptance criteria
    • Returns structured summary: {criteria: [...], scope: "...", risks: [...]}
  3. Call Haiku sub-agent with "read-diffs" task
    • Sub-agent runs git status, git diff
    • Returns list of changed files + file sizes
  4. Call Haiku sub-agent with "read-files" task
    • Sub-agent reads each changed file + context
    • Returns file contents with line numbers
  5. Call Haiku sub-agent with "run-tests" task
    • Sub-agent runs nix develop --command cargo test
    • Returns test results: pass/fail, timing, output excerpt
  6. Call Haiku sub-agent with "analyze-code" task
    • Sub-agent performs preliminary code analysis
    • Returns findings: bugs, regressions, plan mismatches, opportunities
  7. Main Claude synthesizes all sub-agent findings
    • Validate and prioritize findings
    • Make judgment calls on severity
    • Write final feedback.md

4. Haiku Sub-Agent Prompts

Each task type will have a focused system prompt template:

Example: "analyze-code" task prompt 
You are a code reviewer analyzing Rust code for the Oxide project.
Focus on:
1. Bugs and undefined behavior (memory safety, panic paths)
2. Plan mismatches (does this match docs/plan/...)
3. Test coverage gaps
4. Performance regressions

File contents and acceptance criteria will be provided.
Return findings as JSON:
{
  findings: [
    { severity: "issue|suggestion|question", title: "...", file: "...", line?: number, comment: "..." }
  ],
  summary: "brief assessment"
}

Benefits & Tradeoffs

Benefits

  • Faster exploration: Haiku handles file I/O, git commands, test runs with lower latency
  • Preserved context budget: Main Claude has full capacity for complex analysis and synthesis
  • Cost efficiency: Haiku sub-agent is cheaper than main Claude for exploratory tasks
  • Scalability: Can parallelize multiple exploration tasks (read multiple files in parallel via haiku)
  • Better findings: Main Claude can focus on prioritization and judgment calls, not bogged down by I/O

Tradeoffs & Risks

  • Latency: Serial sub-agent calls add round-trip latency (mitigated by async/parallel calls where possible)
  • Model consistency: Haiku may miss subtle issues that Sonnet/Opus would catch (mitigated by main Claude validation step)
  • Complexity: Sub-agent orchestration adds implementation complexity (mitigated by clear task interfaces)
  • Token usage: Passing findings back to main Claude duplicates some context (acceptable tradeoff)
