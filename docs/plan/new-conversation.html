Oxide — Implement /new Slash Command

Start a fresh conversation without clearing the message history

Overview

Add a /new slash command that allows users to start a fresh conversation with the current agent, clearing the conversation context but optionally preserving the chat history for reference.

Behavior

  • User types /new and presses Enter
  • If currently streaming, stop the stream immediately
  • Clear all message history from the visual chat (messages list is emptied)
  • Clear the current conversation ID (set to None)
  • A system message is shown: "Started a new conversation with [agent_name]"
  • Next message will create a fresh conversation_id with the API

Requirements

  • The /new command must be discoverable via tab completion
  • It should work alongside /switch (which changes agents via picker)
  • Must stop any active streaming immediately
  • Must clear all visible messages (including user and agent history)
  • Should support filtering for command menu (e.g., /ne matches /new)
  • Full test coverage for command parsing and action handling using TDD

Design Decisions

  • Full reset: /new clears both the visual history AND conversation context—fresh slate for the user
  • Stream interruption: If streaming is active, stop it to prevent partial responses cluttering the new conversation
  • Simple parsing: Like /switch, parsed as a complete slash command at submit time
  • System message feedback: Confirm the action to the user without ambiguity
  • Enum variant in handler: Add SlashCommand::New for type safety and exhaustiveness checking
  • TDD-first: Write tests before implementation to define expected behavior
